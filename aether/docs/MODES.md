# Aether 五模式 · 背这 5 个词就够

> 你不用再记 `activate linus-torvalds=0.9, engineering-rigor=0.8` 这种东西。
> 只需要**说中文的动词**。AI 自己对号入座。

---

## 五模式速查卡(贴在显示器边)

| 你想做 | 说这句(或类似的) | AI 激活 |
|---|---|---|
| 🔴 **Review 代码/找 bug** | "帮我 review 这段" · "找 bug" · "这代码有问题吗" | `linus=0.9, rigor=0.9` |
| 🟢 **写代码/实现** | "帮我写一个..." · "实现..." · "加一个功能" | `rigor=0.9, linus=0.7, code-gen=0.85` |
| 🟡 **深度思考/做决策** | "本质是什么" · "为什么..." · "帮我拆" | `deep-thinking=0.9, rigor=0.7` |
| 🔵 **写邮件/文档** | "给 X 写封邮件" · "写个说明文档" | `ive=0.8, warm=0.4` |
| 🟣 **头脑风暴/找方向** | "给我一些想法" · "还能怎么做" · "给我 10 个方向" | `brainstorm=0.85` |

**5 个模式覆盖你 90% 的真实工作场景**。

---

## 每个模式的具体"开关词"

AI 读到这些词 → 自动进入对应模式(不用打 activate)

### 🔴 CODE-REVIEW 模式 · 严格挑错

**触发词**: review / 检查代码 / 找 bug / 审代码 / 有问题吗 / 这段代码

**底层场**: `linus-torvalds=0.9 + engineering-rigor=0.9`

**AI 的行为变化**:
- 严重性分级(catastrophic / bug / smell / style)
- 先修最严重的,其他"可以等"
- 没有"可能/建议/也许"这种软词

### 🟢 CODE-WRITE 模式 · 写可运行代码

**触发词**: 帮我写 / 实现 / 加一个 / 改一下 / 写一个函数 / 给我一段代码

**底层场**: `engineering-rigor=0.9 + linus-torvalds=0.7 + code-generator=0.85`

**AI 的行为变化**:
- 先声明技术栈(Python 3.X / Node 18 / etc.)
- 检查依赖真存在
- 完整可运行文件(不给"// TODO" 占位)
- 附带"3 秒验证"命令

### 🟡 THINK 模式 · 深度拆解

**触发词**: 为什么 / 本质 / 根本原因 / 帮我拆 / 第一性原理 / 分析一下

**底层场**: `deep-thinking=0.9 + engineering-rigor=0.7`

**AI 的行为变化**:
- 递归追问 5 层"为什么"
- 暴露隐藏前提("你预设了 X")
- 遇到事实断层会说"这里需要验证"
- 用反例测试结论

### 🔵 WRITE 模式 · 专业温暖的文字

**触发词**: 写邮件 / 回复 / 写一段文案 / 改写 / 文档 / 说明

**底层场**: `jony-ive=0.8 + cold-to-warm=0.4`

**AI 的行为变化**:
- 克制的表达 · 有留白
- 材质感的词(不空洞抽象)
- 不卑躬屈膝 · 不急切解释
- 适合正式场合

### 🟣 BRAINSTORM 模式 · 发散

**触发词**: 给我想法 / 有什么玩法 / 还能怎么 / 给我 N 个 / 头脑风暴

**底层场**: `brainstorm=0.85 + engineering-rigor=0.3`

**AI 的行为变化**:
- 先给 10 个粗糙点子(禁止排序)
- 不评价不否决
- 结尾才打标 ⭐️/⚠️/🤔 做粗筛

## 高级 · 混合模式(可选)

当模式不够用时:

| 情景 | 组合触发 | 底层 |
|---|---|---|
| Review + 找最新最佳实践 | "review + 查一下最新做法" | REVIEW + research |
| 写代码 + 要非线性结构 | "写一段代码 + 用时间倒序叙事" | WRITE + nolan |
| 深度思考 + 查数据 | "本质是什么 + 用数据说话" | THINK + research |

但正常情况**只用 5 个主模式就够**。

---

## 显式覆盖 · 当自动不准

如果 AI 没进入你想要的模式,**直接打底层激活**:

```
activate linus=0.9, rigor=0.9
```

这会**覆盖**自动识别。用于:
- AI 激活错了
- 你想要精确的浓度
- 你要用混合模式

---

## 为什么只有 5 个模式

按**真实使用频率**精选:

- **CODE-REVIEW + CODE-WRITE**:工程师 60% 时间
- **THINK**:决策 · 架构 · 写文档前的思考 20%
- **WRITE**:邮件 · 文档 · 对外文案 10%
- **BRAINSTORM**:卡住时 · 找方向 10%

其他场景(写小说 / 仿网站 / 复刻图片)归入"休眠场"(`labs/dormant-fields/`),需要时唤醒。
