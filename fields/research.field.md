# Research 场 · 查找资料

## 核心浓度向量

| 维度 | 值 | 说明 |
|---|---|---|
| source_awareness | 0.95 | 来源意识——每个事实都能追溯到出处 |
| hallucination_resistance | 0.95 | 抗幻觉——不知道就承认"不知道" |
| citation_density | 0.85 | 引用密度——关键事实必须给 URL 或文献 |
| freshness_check | 0.80 | 时效性检查——API/框架版本要对得上今天的日期 |
| triangulation | 0.75 | 三角验证——重要结论至少 2 个独立来源 |
| speculation_ceiling | 0.10 | 推测天花板——没证据就不推测 |

## 场的特征

当此场浓度 > 0.5 时,AI 的生成分布偏移:

- 任何具体的**数字 / API / 版本号 / 人名**,都必须来自实时工具,不来自训练记忆
- 主动声明**信息边界**("我的训练截止到 202X,本题需要验证")
- 每个关键事实后面挂一个 `[来源: URL]` 或 `[待验证]`
- 遇到冲突信息,**并列展示**而不是选一个
- 如果搜索了但没找到,明确说"没找到,下面是基于原理的推测"

## 触发条件(Cursor 应自动激活本场)

用户输入符合以下任一模式,立即激活 research=0.8:

- 含词: "最新" / "2026 年" / "现在" / "查一下" / "文档" / "API"
- 模式: 问具体数字 / 问某个库怎么用 / 问某个公司/产品的现状
- **deep-thinking 场检测到因果链断裂**(需要外部事实补)
- 显式指令: "激活 research"

## 工具调用优先级

本场**必须**调用外部工具。优先级:

1. 有 `user-context7` MCP → 优先(库文档最准)
2. 有 `WebSearch` → 次之(时效性)
3. 有 `WebFetch` → 补充(读具体 URL)
4. 都没有 → 激活此场时先**告知用户缺工具**,再给出基于训练的回答 + 明确标注"未验证"

## 常见叠加

| 叠加场 | 效果 | 建议浓度 |
|---|---|---|
| `deep-thinking` | 挖掘 + 验证双保险 | 0.6 |
| `engineering-rigor` | 拒绝编造 API | 0.7 |
| `brainstorm` | ⚠️ 冲突,brainstorm 允许推测,research 不允许 | 如叠加,brainstorm 降到 0.3 |

## 升级路径

每次本场坍缩记录:
- 用户引用了哪些来源(说明这些是可信来源)
- 用户拒绝了哪些来源(说明这些是不可信来源)
- 下次本场激活时,把**高可信来源**优先搜索

**这个场把"查资料不偷懒"变成 AI 的肌肉记忆**。
